# Python 并发编程对比

## 三种并发模型

Python 提供三种主要的并发编程方式：

| 模型 | 适用场景 | GIL 影响 |
|------|---------|---------|
| threading | I/O 密集型 | 受 GIL 限制 |
| multiprocessing | CPU 密集型 | 不受 GIL 限制 |
| asyncio | I/O 密集型 | 不受 GIL 限制 |

## threading — 多线程

CPU 密集任务在多线程下不但不能加速，反而因为 GIL 竞争和上下文切换比单线程更慢。这就是著名的"GIL 性能瓶颈"。但 I/O 密集型任务（网络请求、文件读写）可以使用多线程获得并发收益。

## multiprocessing — 多进程

每个进程有独立的 GIL，真正并行执行。缺点是进程创建开销大，进程间通信（IPC）比线程共享内存复杂。适合 CPU 密集型、数据独立的场景。

## asyncio — 异步协程

asyncio 是最现代的方案。单线程事件循环，通过协程切换实现并发。asyncio 不是用多线程实现并发，而是用"协作式多任务"——协程在 await 处主动让出控制权，事件循环切换到下一个就绪的协程。

## 选择指南

如果任务主要是 I/O 密集型（网络、文件、数据库），优先选 asyncio（大量并发）或 threading（简单场景）。如果是 CPU 密集型，数据独立用 multiprocessing，需要共享大量数据用 Numba 或 C 扩展。
